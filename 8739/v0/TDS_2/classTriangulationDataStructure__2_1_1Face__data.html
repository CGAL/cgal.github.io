TriangulationDataStructure_2::Face_data Concept Reference
2D Triangulation Data Structure Reference » Concepts

Definition

Various algorithms using a triangulation data structure, such as Delaunay triangulations or Alpha Shapes, must be able to associate a state to a face elemental. For efficiency, this information must be stored directly within the face.

This class is only meant to store a state (Boolean). Consequently, the state must be the default value (i.e. false) unless a setting function (mark_in_conflict(), etc.) has been called.

The three states are "in conflict", "on boundary", and "processed". By default, a face is not in conflict, not on boundary, and not processed.
